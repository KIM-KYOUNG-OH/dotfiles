# Plan: AI시대-신입-개발자-생존법 글 구조 조정

## 변경된 "위기의 숫자들"의 핵심 메시지

1. **역사적 관점**: 1950년대부터 지금까지 "개발자 대체" 예측은 항상 있었지만, 실제로는 생산성 도구가 됨
2. **학습 능력 강조**: "얼마나 알고 있는지"보다 "얼마나 잘 배울 수 있는지"가 중요
3. **위기=기회**: 새로운 규칙을 이해하는 사람에게는 기회
4. **글의 목적 명시**: AI를 활용하는 개발자로 성장하는 방법 제시

## 현재 구조와 문제점

| 섹션 | 현재 내용 | 문제점 |
|------|----------|--------|
| 위기의 숫자들 | ✅ 확장됨 - 역사, 학습능력 강조 | 없음 |
| AI가 바꾼 게임의 규칙 | Specification-Translation-Verification | "학습" 메시지와 연결 약함 |
| 코드는 전체의 10-20% | Bus factor, Jack Reeves | 괜찮음 |
| AI가 못하는 것을 하라 | Low/High Context | 괜찮음 |
| 문제 분해와 검증 | Kent Beck 문제 분해 | "학습 능력" 연결 가능 |
| 결론 | 주니어에게/회사에게 | "학습" 메시지 강화 필요 |

## 조정 계획

### 1. "AI가 바꾼 게임의 규칙" 섹션 도입부 연결
- 현재: "그렇다면 새로운 규칙은 무엇일까?"
- 변경: "위기의 숫자들"에서 언급한 "새로운 게임의 규칙"과 자연스럽게 연결
- 역사적 맥락(과거의 대체 예측들)을 받아 현재 AI의 진짜 영향으로 연결

### 2. "문제 분해와 검증" 섹션에 학습 관점 추가
- Kent Beck의 주니어→시니어 성장 과정을 "학습 능력"과 연결
- "복잡성을 다루는 방법을 배우면서" 성장한다는 메시지 강화

### 3. "결론" 섹션 대폭 강화
- 현재: 역량 나열 위주
- 변경: **"잘 배우는 능력"을 보여주는 구체적 방법** 제시
  - AI를 학습 도구로 활용하는 구체적 방법
  - knowledge doubling curve 맥락에서 지속적 학습의 중요성

### 4. 전체 흐름 재정립
```
위기의 숫자들 (역사 + 학습능력 강조)
    ↓ "그렇다면 규칙이 뭔가?"
AI가 바꾼 규칙 (S-T-V + AI 역설)
    ↓ "코딩만이 아니라는 증거"
코드는 10-20% (실제 경험)
    ↓ "그럼 뭘 해야 하나?"
AI가 못하는 것 (High Context)
    ↓ "구체적으로 어떤 역량?"
문제 분해와 검증 (Kent Beck + 학습 관점)
    ↓ "결론"
결론 (잘 배우는 능력을 보여주는 법)
```

## 구체적 수정 내용

### 수정 1: "AI가 바꾼 게임의 규칙" 도입부 (40-44행)
**현재:**
> 그렇다면 새로운 규칙은 무엇일까?
> 많은 사람들이 "AI가 코딩을 대신해준다"고 생각한다.

**변경:**
> 역사가 반복된다면, AI도 개발자를 대체하기보다 생산성을 높이는 도구가 될 것이다. 하지만 규칙은 분명히 바뀐다. 그렇다면 새로운 규칙은 무엇일까?
> 많은 사람들이 "AI가 코딩을 대신해준다"고 생각한다.

### 수정 2: "문제 분해와 검증" 섹션 (108-112행)
Kent Beck 인용 뒤에 **학습 관점** 추가:
> 주니어는 처음에 복잡성을 다루는 방법을 모른다. 하지만 **트릭과 기술을 배우면서** 점점 복잡한 것을 다루게 된다. 중요한 것은 이 과정이 **학습**이라는 점이다.

### 수정 3: "결론 - 주니어에게" 섹션 (126-137행)
**현재의 간결한 조언(TDD, 도메인, 검증)을 유지하면서 "잘 배우는 능력" 관점 추가:**

현재:
> 핵심 역량은 두 가지다.
> - Specification: 문제를 정의하고 분해하는 능력
> - Verification: 결과를 검증하고 판단하는 능력

변경:
- "위기의 숫자들"에서 언급한 "얼마나 잘 배울 수 있는지"와 연결
- 구체적 실천법 추가:
  - AI를 선생님으로 활용 (코드 설명 요청, 왜 이렇게 했는지 질문)
  - 작은 프로젝트로 Spec-Trans-Verify 전체 사이클 경험
  - 배운 것을 정리하고 공유 (블로그, 사내 위키 등)
