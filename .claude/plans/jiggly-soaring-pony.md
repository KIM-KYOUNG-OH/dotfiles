# Teams CS 채널 → 마크다운 가이드 문서 생성 계획

## 목표
Teams "🛠시스템개발 x CS" 채널의 CS 대응 내역을 수집하여, 개발팀이 빠르게 참고할 수 있는 가이드 문서 생성

## 핵심 요구사항
1. **중복 제거**: 동일/유사한 CS는 하나로 통합
2. **빈도 기반 우선순위**: 자주 발생하는 CS를 상단에 배치
3. **기간별 문서 분리**: 월 단위 (데이터 많으면 주/일 단위)
4. **저장 위치**: `docs/cs-guide/`

## 출력 문서 구조

```
docs/cs-guide/
├── INDEX.md                    # 전체 목차 + 빈도 순위
├── 2026-01/                    # 월별 폴더
│   ├── 2026-01-CS-가이드.md    # 월별 CS 정리
│   └── ...
├── 2025-12/
│   └── 2025-12-CS-가이드.md
└── ...
```

### 문서 포맷 (월별)

```markdown
# 2026-01 CS 대응 가이드

## 요약
- 총 CS 건수: N건
- 주요 도메인: 주문(X건), 상품(Y건), ...

## 빈도순 CS 목록

### 1. [주문] 환불불가 체크 해제 요청 (5건)
**증상**: 팬클럽 상품에 환불불가가 체크되어 있어 환불 요청 버튼 비활성화
**원인**: 상품 자체에 환불불가 설정
**대응**: 해당 상품의 환불불가 체크 해제
**관련 날짜**: 01/27, 01/25, 01/20, ...

### 2. [배송] 배송상태 수정 요청 (3건)
...
```

## 실행 단계

### Phase 1: 데이터 수집
1. Teams 채널 하단(최신글)으로 스크롤
2. 각 게시글에 대해:
   - "더보기" 클릭하여 전체 내용 확인
   - "회신 n개 열기" 클릭하여 댓글 수집
   - 날짜, 작성자, 내용, 첨부파일 정보 추출
3. 위로 스크롤하며 반복 (전체 채널)

### Phase 2: 분류 및 그룹핑
1. 각 CS를 도메인별로 태깅 (주문/배송/상품/회원/결제/기타)
2. 유사한 CS 패턴 식별 및 그룹핑
3. 빈도수 카운팅

### Phase 3: 문서 생성
1. 월별로 데이터 분리
2. 각 월별 문서 생성 (빈도순 정렬)
3. INDEX.md 생성 (전체 요약 + 링크)

## Playwright 작업 흐름

```
1. 채널 페이지 로드 확인
2. 채널 메시지 영역 하단으로 스크롤
3. 반복:
   a. 현재 보이는 게시글 목록 캡처
   b. 각 게시글에 대해:
      - "더보기" 있으면 클릭
      - "회신 n개 열기" 있으면 클릭
      - 내용 추출
   c. 위로 스크롤
   d. 새로운 게시글이 없으면 종료
4. 수집 데이터 분류 및 문서 생성
```

## 예상 출력 예시

### INDEX.md
```markdown
# CS 대응 가이드 INDEX

## 전체 통계
- 수집 기간: 2024-01 ~ 2026-01
- 총 CS 건수: XXX건

## 빈도 TOP 10
1. [환불불가 체크 해제] - 15건 → [상세](./2026-01/...)
2. [배송상태 수정] - 12건
...

## 월별 문서
- [2026-01](./2026-01/2026-01-CS-가이드.md) - N건
- [2025-12](./2025-12/2025-12-CS-가이드.md) - M건
...
```

## 검증 방법
1. 생성된 문서가 `docs/cs-guide/` 폴더에 존재하는지 확인
2. INDEX.md에 모든 월별 문서 링크가 있는지 확인
3. 빈도순 정렬이 올바른지 확인
4. 샘플로 2-3개 CS 내용이 원본과 일치하는지 대조

## 주의사항
- Teams 세션 타임아웃 시 재로그인 필요
- 대용량 채널의 경우 여러 세션에 걸쳐 작업 필요
- 첨부 이미지는 텍스트 설명으로 대체
