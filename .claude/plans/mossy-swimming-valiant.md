### 3. Chapter 17: AIs, LLMs, and God Knows What (Robert Martin)
- **위치**: `003-RESOURCES/`

- **위치**: `003-RESOURCES/`
- **핵심 내용**:

  - 제본스의 역설: 효율화 → 더 많은 작업 → 더 많은 인력

## 역사적 관점: 개발자 대체 시도의 반복되는 실패

### 프로그래밍 언어 진화와 "개발자 불필요" 예언의 역사

| 시대 | 기술 | "개발자 필요 없다" 주장 | 실제 결과 |
|------|------|------------------------|----------|
| 1954 | FORTRAN | 폰 노이만의 거부 | 고급 언어 혁명 시작 |
| 1962 | COBOL | "누구나 코딩 가능" | Grace Hopper의 반박 |
| 1972 | C | "최종 언어" | 생산성 향상, 수요 증가 |
| 1985 | C++ | "최종적 프로그래밍 언어" | OOP 확산, 더 많은 프로젝트 |
| 1992 | Visual Programming | "드래그 앤 드롭이 미래" | 한계 노출, 코더 여전히 필요 |
| 2008 | Cloud | "엔지니어 불필요" | DevOps 탄생, 역할 확대 |
| 2025 | GenAI | "이번이야 진짜" | ??? |

### 패턴의 분석

1. **새로운 추상화 기술 등장** → 효율성 향상
2. **"개발자 대체" 예언** → 두려움 확산
3. **실제 결과**: 추상화 수준만 증가, 복잡도 유지
4. **제본스의 역설**: 효율화 → 더 많은 프로젝트 → 더 많은 개발자 필요

### AI: 또 다른 추상화 단계

- AI는 FORTRAN, COBOL, C++, 클라우드와 같은 선상에 있음
- 코드 생성 자동화 ≠ 개발자 대체
- 개발자 역할의 진화: 코더 → 아키텍트 → "AI의 변호사"

> "AI가 당신의 일자리를 빼앗을까요? 절대 아닙니다. 만약 당신이 진화한다면요."
> — Werner Vogels, AWS CTO
