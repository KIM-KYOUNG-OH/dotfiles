# AI 활용 기법 문서 수집 프로젝트 계획

## 프로젝트 개요

**목표**: `~/DocumentsLocal/msbaek_vault/003-RESOURCES/AI` 폴더의 286개 문서를 분석하여 AI 활용 기법을 체계적으로 수집

**사용자 선택 옵션**:
- 추출 범위: **핵심 기법만** (프롬프트 패턴, 워크플로우, 도구 설정 등 바로 적용 가능한 구체적 기법)
- 결과 형식: **표(Table)** (기법명|설명|적용상황|출처)
- 실행 방식: **테스트 후 진행** (첫 30개 처리 후 검증)

**대상 폴더 구조**:
- 루트: 58개 문서
- `/tools`: Claude Code, MCP, Cursor 등 도구 관련
- `/Usage`: AI 활용 사례 및 워크플로우
- `/MCP`: Model Context Protocol 관련
- `/CLAUDE-CODE`: Claude Code 전용
- `/PROMPTS`, `/MY_PROMPTS`: 프롬프트 관련
- `/LIMITATIONS`: AI 한계 및 주의사항
- `/best-practices`: 모범 사례
- 기타: `/BDD`, `/Pair`, `/work-log`, `/Claude`

## 실행 단계

### Phase 1: 초기 설정 (1회)

1. **Backlog.md 마스터 태스크 생성**
   ```
   태스크: "AI 활용 기법 문서 286개 분석 및 수집"
   - 하위 태스크로 배치별 작업 관리
   ```

2. **진행 상황 추적 파일 생성**
   - `_COLLECTION_PROGRESS.md` 생성
   - 전체 286개 문서 목록 기록

3. **결과 저장 파일 생성**
   - `_AI_TECHNIQUES_COLLECTION.md` (최종 결과)
   - `_BATCH_RESULTS/` 폴더 (배치별 중간 결과)

### Phase 2: 첫 배치 테스트 (검증)

**배치 크기**: 30개 문서 (10개 × 3 병렬 에이전트)

1. **테스트 배치 실행**:
   - 첫 30개 문서만 처리
   - 결과 품질 확인 후 사용자 승인

2. **Analyzer Agents 병렬 실행** (Task tool 사용):
   ```
   subagent_type: "general-purpose"
   run_in_background: true

   각 에이전트 프롬프트:
   - 할당된 10개 문서 읽기
   - 핵심 기법만 추출 (바로 적용 가능한 구체적 기법)
   - 표 형식으로 정리: | 기법명 | 설명 | 적용 상황 | 출처 |
   - 추상적 개념이나 일반론은 제외
   ```

3. **테스트 결과 검토**:
   - 추출된 기법 품질 확인
   - 사용자 피드백 반영
   - 문제 없으면 전체 진행

### Phase 3: 전체 배치 처리 (반복, 테스트 승인 후)

1. **Coordinator 작업**:
   - 진행 상황 파일에서 다음 처리할 30개 문서 확인
   - 10개씩 3개 그룹으로 분할
   - 병렬 에이전트 실행 및 결과 수집

2. **결과 수집 및 저장**:
   - 3개 에이전트 결과 대기 (TaskOutput)
   - `_BATCH_RESULTS/batch_N.md`에 저장
   - 진행 상황 파일 업데이트 (완료된 문서 체크)

### Phase 4: 통합 및 정리 (마지막)

1. **Aggregator Agent 실행**:
   - 모든 배치 결과 파일 읽기
   - 중복 기법 제거 및 병합
   - 카테고리별 분류
   - 최종 `_AI_TECHNIQUES_COLLECTION.md` 작성
